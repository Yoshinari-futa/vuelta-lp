# VUELTA LP 公開手順（詳細版）

## 📊 現在の状況

✅ **プロジェクトは既にVercelに存在しています**
- プロジェクト名: ブエルタ・LP
- URL: `vuelta-lp.vercel.app`
- GitHubリポジトリ: `吉成フタ/vuelta-lp` に接続済み
- ⚠️ 警告アイコンが表示されている = 最新の変更が反映されていない可能性

## 🚀 最新の変更を公開する方法

### 方法1: GitHubにプッシュして自動デプロイ（推奨）

プロジェクトは既にGitHubと連携されているので、GitHubにプッシュするだけで自動的にデプロイされます。

#### 手順：

1. **GitHubにプッシュ**
   ```bash
   cd /Users/yoshinarifuta/lp
   git push origin main
   ```

2. **認証が必要な場合**
   - GitHubのPersonal Access Tokenを使用するか
   - GitHub Desktopなどのアプリを使用してください

3. **自動デプロイの確認**
   - Vercelダッシュボードの「展開」（Deployments）タブを開く
   - 新しいデプロイが自動的に開始されます
   - 2-3分で完了します

---

### 方法2: Vercelダッシュボードから手動で再デプロイ

GitHubにプッシュできない場合：

1. **Vercelダッシュボードでプロジェクトを開く**
   - 「ブエルタ・LP」をクリック

2. **「展開」（Deployments）タブを開く**

3. **最新のデプロイを確認**
   - 警告アイコンの原因を確認
   - エラーメッセージがあれば確認

4. **手動で再デプロイ**
   - 「...」（三点リーダー）メニューから「再デプロイ」を選択
   - または、GitHubリポジトリの最新コミットを選択して再デプロイ

---

### 方法3: ローカルの変更を直接デプロイ

Vercel CLIを使用（認証が必要）：

1. **ログイン**
   ```bash
   npx vercel login
   ```

2. **デプロイ**
   ```bash
   npx vercel --prod
   ```

---

## 🔍 警告アイコンの確認方法

1. **Vercelダッシュボードでプロジェクトを開く**
2. **「展開」（Deployments）タブを開く**
3. **最新のデプロイをクリック**
4. **ビルドログを確認**
   - エラーメッセージがあれば確認
   - 警告があれば内容を確認

---

## ✅ デプロイ後の確認

1. **URLにアクセス**
   - `https://vuelta-lp.vercel.app` にアクセス
   - または、Vercelダッシュボードに表示されているURL

2. **確認項目**
   - [ ] ホームページ（`/`）が正常に表示される
   - [ ] リクルートページ（`/recruit`）が正常に表示される
   - [ ] モバイル表示が正常に動作する
   - [ ] フォントが正しく読み込まれている
   - [ ] 画像が正しく表示される

---

## 🆘 問題が発生した場合

### ビルドエラーが出る場合
1. Vercelダッシュボードの「展開」タブでビルドログを確認
2. エラーメッセージを確認
3. 必要に応じて修正して再デプロイ

### 警告アイコンが消えない場合
1. 「展開」タブで最新のデプロイの状態を確認
2. 成功している場合は、警告は無視しても問題ありません
3. 失敗している場合は、エラーログを確認して修正

---
